## 요약

#2070에서 후속 이슈로 분리했던 **본문 NO_LS 실폭 래핑** 탐사에서 출발해, 그 과정에서

PDF의 괘선·글리프·좌표 직독, 통제 사다리)로 축 단위 분해·정합시켰다.

**본문 문단 reflow 확장 자체는 최종 기각**(stage68): 후속 축들이 게이트 정합을 대체

가 담당한다. 최종 채택분은 **셀 순수 빈 문단 합성 + 파생 축 세트**다.

결과: **76076=82 / 86712=65 복귀**(#2070 잠정치 83/64 해소), **sijang=307 밀도 핀
정렬**(재병합 후 -22 회귀를 stage65~68 이분으로 전량 해소), 기존 게이트(80168=157,

## 확정 축 (구현 세트)

각 축의 실측 근거는 stage 문서에 기록. 오라클 없는 규칙은 넣지 않았다(실측주의).

### 로드(IR) — NO_LS 재계산 확장
1. **셀 순수 빈 문단 합성 `include_cell_empty`** (`document_core/commands/document.rs`):
   HWP5 native의 NO_LS 확장은 **셀 내부의 컨트롤 없는 순수 빈 문단 한정** —
   - 본문 확장은 **기각**(stage68): 빈 문단은 em 폴백(#2070 축3)이 담당(합성 시 pi424
     오라클 위반·156 회귀), 텍스트 문단은 composer recompose 가 담당(합성 lineseg 는
     흐름 소비 문단당 ~2.7px 팽창 — sijang 밀도 핀 -5쪽의 전원인).
   - 셀 텍스트 문단·컨트롤(중첩 표 등) 호스트도 제외 — 합성 시 80168 pi=1243 행6
     이중 계상(264→467px).
   - **사선(대각선) 셀 제외**(셀 BF+cellzone): 코너 라벨의 짝 빈 문단은 흐름 비배치
     (21761835 r0 선언 52.4px 유지, #2146).
   - 셀 reflow 폭은 실효 pad 규칙(아래 4)과 통일 — 줄수 이원화 제거.

### 측정(높이·폭)
2. **anchor 몫0 정밀화** (`height_measurer.rs`, #2169 확장): 중첩 표 anchor 빈 문단
   몫0은 **표 단독 컨트롤** 앵커 한정. 표+타 컨트롤(누름틀 필드) 동반 앵커는 em×ls
   줄박스 계상 — 86712 근거설명 괘선 회계(호스트 15pt×ls120=24px)와 80168 몫0 정답의
   교차 판별. **86712=65의 결정타.**
3. **미설치 폰트 전각 폴백** (`text_measurement.rs` kopub_char_width): KoPub돋움 한글
   0.84→**1.0em**. 근거: KoPub 미설치 확인 + 한컴 PDF 글리프 직독(문서 전역 Haansoft
   Batang 치환, 12pt 한글 16px 전각) + 폰트 사다리(전 폰트 줄피치 = fs×ls%, 메트릭 가설
   기각). 86712 표172 −389px 과소의 주성분(+11줄급) 해소.
   ※ 환경 의존성: 이 규칙은 "미설치→치환" 환경(본 게이트 오라클 환경)의 실동작이다.
   KoPub 설치 환경에선 실폭이 좁아질 수 있음 — 단위테스트 핀에 근거 주석 명시.
4. **pad 실효 — 전축0 미지정 규칙** (`model/table.rs`, `table_layout.rs`): aim=false여도
   표 기본 pad가 **네 축 모두 0**(미지정)이면 셀 저장 pad 사용. #1785 위생 한도
   (≥2500HU 보존값은 표 기본) 유지. pad 사다리의 "표 기본" 실측(일부 축만 0)과 구분.
   exam_social 글상자(#1100)·86712 구분선 괘선 실측의 공통 정합.

### 페이지네이션(분할·간격)
5. **자리차지 표 outer margin** (`typeset.rs`, #1841 확장): TopAndBottom 표의
   outer_margin top을 전면 계상, bottom은 빈 호스트도 계상하되 —
   **호스트 저장 lineseg 보유 시 fit 제외(흐름 전용, `after_for_fit` 분리)**,
   NO_LS 호스트는 fit 포함. 판별 근거: hwpspec(저장 문서) 178쪽 핀 vs 86712(생성계)
   표182 분할 실측 — #2195 전체의 "저장 보존 vs NO_LS 재계산" 원칙과 동일 축.
6. **1×1 중첩 표 fragment 임계 완화** (`table_layout.rs`, #2007): 2페이지→1페이지.
   과거 임계 사유(fragment 경계 ±1 회귀)는 래핑·측정 정합으로 소멸 — 76076 r6
   통이월(838px 공백) 해소, **76076=82의 결정타**.
7. **중첩 표 스트레치 TAC 배제** (`rendering.rs` render_normalized): 스트레치(내부 표를
   부모 셀 전폭으로 확장)는 비-TAC 한정. TAC(글자처럼) 중첩 표는 인라인 개체 —
   hcar-001 4×1 동의 표 한컴 PDF 590.7px = 원폭 유지(#1195).
