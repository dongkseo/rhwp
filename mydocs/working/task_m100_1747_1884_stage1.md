# Task M100 #1747/#1884 Stage 1

## 범위

- #1747: `export-pdf`에서 generic fallback font family를 CLI 옵션으로 지정할 수 있게 한다.
- #1884: `export-pdf`에서 수식 SVG font-family의 우선 폰트를 CLI 옵션으로 지정할 수 있게 한다.

두 이슈는 모두 PDF export의 fontdb/수식 font-family 설정 문제이므로 같은 브랜치에서 처리한다.

## 구현 방향

- `src/renderer/pdf.rs`에 PDF 전용 폰트 옵션 구조체를 추가한다.
- 옵션 미지정 시 OS별 기본 fallback family를 사용한다.
- 사용자가 지정한 family가 PDF 변환 fontdb에 없으면 warning을 출력한다.
- 기존 `svgs_to_pdf` API는 유지하고, 옵션을 받는 변형 API를 추가한다.
- `export-pdf` CLI에 fallback serif/sans/mono, 수식 폰트, 폰트 탐색 경로 옵션을 연결한다.

## 검증 계획

- `cargo fmt --check`
- `cargo test --lib renderer::pdf`
- `cargo run --bin rhwp -- export-pdf --help` 성격의 도움말/파싱 경로 확인

## Stage 1 결과

- `PdfExportOptions`를 추가하고 기존 `svgs_to_pdf` API는 default 옵션 경로로 유지했다.
- `export-pdf`에 `--fallback-serif`, `--fallback-sans`, `--fallback-mono`, `--equation-font`, `--font-path`를 추가했다.
- 옵션 미지정 시 OS별 기본 fallback family를 사용하도록 했다.
- 최종 선택된 fallback family와 수식 우선 폰트가 fontdb에 없으면 warning을 출력하도록 했다.
- PDF 변환 직전 SVG 후처리에서 한글 fallback과 수식 font-family를 반영하도록 했다.
- 공백 포함 경로/폰트명 예시는 큰따옴표(`"..."`) 기준으로 도움말과 매뉴얼을 정리했다.
