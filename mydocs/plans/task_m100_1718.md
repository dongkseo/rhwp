# Task #1718 수행계획서 — 대형 RowBreak 셀 over-fill under-pagination 수정

## 배경
hwpdocs 3,249건 rhwp vs 한글(OLE) 페이지·PI 회귀 비교에서 대형 표 under-pagination |Δ|≥2 케이스 8건 관측. 최악 −8쪽:
`[별표 27] 경사형 휠체어리프트 안전기준(승강기안전부품 안전기준 및 승강기 안전기준).hwp` — rhwp 40쪽 vs 한글 48쪽.

## 원인 (정량 규명)
- 전체가 단일 5×1 RowBreak 표, 마지막 거대 셀(654문단/~1,200라인).
- 기본 줄 pitch 19.2pt = rhwp 1920 HU **동일**, 한글 페이지 하단까지 꽉 채움, 그림 크기 차이 아님 — 모두 배제.
- 확정: `advance_row_cut`/`advance_row_block_cut`(table_layout.rs)의 `visible_tail_before_spacer` grace 가
  `units[j+1..].iter().any(|u| u.empty_spacer)` — **뒤에 spacer 가 하나라도** 있으면 `avail+120px`까지 오버플로 라인을 수용.
  654문단 셀엔 빈 문단 spacer 가 흩어져 있어 **연속 텍스트에서도 항상 grace 적용** → 39줄 대신 40~44줄 과충전, 37쪽 누적 −8쪽.

## 수정 방향
grace 를 **진짜 꼬리줄**(뒤가 전부 spacer)에만 적용하도록 `.any()` → `.all()` 로 정정.
- 연속 텍스트: 뒤에 가시 라인 있음 → `all()` false → 정상 capacity break(39줄) → 한글 정합.
- 진짜 tail-before-spacer(별표1/별표4 over-pagination 방지 케이스): 뒤 전부 spacer → `all()` true → grace 유지 → 무회귀.

## 회귀 게이트 (필수)
1. 대표 파일 dump-pages 40 → ~48쪽 (한글 48 근접).
2. `samples/byeolpyo1`(4쪽), `samples/byeolpyo4`(27쪽) 무회귀 ([[task1658_verification_samples]] 게이트).
3. `cargo test`(advance_row_cut 단위테스트 포함) 무회귀.
4. verify_pi_page_vs_hangul 샘플 재실행 — MATCH율 무회귀, 대형 |Δ|≥2 케이스 개선 확인.
