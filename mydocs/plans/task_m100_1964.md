# Task M100 #1964 계획 - raw IME/iOS 입력 page-local refresh 안전 가드

## 이슈

- GitHub Issue: https://github.com/edwardkim/rhwp/issues/1964
- 제목: `#1918 후속: raw IME/iOS 입력 경로 page-local refresh 안전 가드 보강`

## 원인 가설

`rhwp-studio/src/engine/input-handler.ts`의 command 기반 텍스트 입력 경로는
`shouldUsePageLocalRefresh()` 호출 시 `insertedText`, `beforePageIndex`, `afterPageIndex`를 넘긴다.
반면 raw IME/iOS 입력 경로는 `afterTextInputEdit(beforePos, afterPos)`만 호출해 줄바꿈/탭, 긴 입력,
page 이동 상황에서 page-local refresh로 잘못 빠질 수 있다.

## 구현 방향

- `afterTextInputEdit`가 `PageLocalTextEditOptions`를 받도록 확장한다.
- IME 조합 중 raw 입력 경로는 실제 조합 문자열과 전후 page index를 전달한다.
- iOS fallback 경로도 현재 textarea 값과 전후 page index를 전달한다.
- 기존 `isPageLocalTextEditCommand` 계약을 재사용하고 별도 분기를 만들지 않는다.
