# Task M100 #1672 수행 계획서 — 2025 행정업무운영 편람 페이지 수 정합

## 목표

`2025 행정업무운영 편람(최종)` 샘플의 전체 페이지 수를 한컴 2024 PDF 기준과 맞춘다.

- GitHub Issue: #1672
- 대상 HWP: `samples/2025 행정업무운영 편람(최종).hwp`
- 대상 HWPX: `samples/2025 행정업무운영 편람(최종).hwpx`
- PDF oracle: `pdf/2025 행정업무운영 편람(최종)-2024.pdf`
- 한컴 PDF 기준: 383쪽
- 초기 rhwp 기준: HWP 416쪽, HWPX 416쪽
- 2026-06-30 중간 구현 후 rhwp 기준: HWP 388쪽, HWPX 390쪽

## 재현 결과

확인 명령:

```bash
pdfinfo 'pdf/2025 행정업무운영 편람(최종)-2024.pdf'
./target/release/rhwp info 'samples/2025 행정업무운영 편람(최종).hwp'
./target/release/rhwp info 'samples/2025 행정업무운영 편람(최종).hwpx'
RHWP_USE_PAGINATOR=1 ./target/release/rhwp info 'samples/2025 행정업무운영 편람(최종).hwp'
```

관찰:

- PDF는 Hancom PDF / Hwp 2024 13.0.0.3622 산출물이며 총 383쪽이다.
- 기본 `TypesetEngine` 경로는 HWP/HWPX 모두 416쪽이다.
- `RHWP_USE_PAGINATOR=1` 경로는 HWP/HWPX 모두 400쪽이다.
- PDF 383쪽의 최종 발행정보 텍스트는 rhwp 416쪽과 대응한다. 즉 파일 불일치가 아니라 중간 페이지네이션 누적 오차다.

중간 구현 후 재확인:

- HWP 기본 `TypesetEngine` 경로: 388쪽
- HWPX 기본 `TypesetEngine` 경로: 390쪽
- HWP 기준 남은 차이: +5쪽
- HWPX 기준 남은 차이: +7쪽

## 1차 원인 후보

### 후보 A — RowBreak 표 분할 과대

후반부에서 `PartialTable`만 있는 페이지가 다수 나타난다.

- 기본 경로 page 383: 구역 11, 문단 3, 103행×2열 RowBreak 표의 continuation
- 기본 경로 page 400: 구역 12, 문단 12, 11행×2열 RowBreak 표의 continuation
- 최종 page 416: PDF 383쪽과 같은 발행정보 페이지

구역별 페이지 수 비교:

| 구역 | TypesetEngine | Paginator |
|------|--------------:|----------:|
| 0 | 8 | 8 |
| 1 | 19 | 18 |
| 2 | 108 | 100 |
| 3 | 18 | 18 |
| 4 | 21 | 22 |
| 5 | 17 | 18 |
| 6 | 10 | 10 |
| 7 | 54 | 53 |
| 8 | 6 | 6 |
| 9 | 32 | 27 |
| 10 | 38 | 37 |
| 11 | 65 | 64 |
| 12 | 19 | 18 |
| 13 | 1 | 1 |

기본 경로의 추가 16쪽은 주로 구역 2, 9, 10~12에서 발생한다. Paginator도 PDF 기준보다 17쪽 많으므로, 공통 표 측정/분할 축과 Typeset 전용 축을 나눠 봐야 한다.

### 후보 B — 셀 내부 `LINE_SEG vpos` 리셋 과대 해석

문제 후보 RowBreak 표의 셀 안 문단에는 `vpos=0` 리셋이 반복된다. #1488에서 이미 빈 오버레이 문단의 vpos 리셋을 hard-break로 과대 해석해 거의 빈 continuation 페이지를 만든 전례가 있다. 이번 샘플은 빈 오버레이뿐 아니라 긴 법령 비교표 셀 내부 텍스트에도 리셋이 섞여 있어, 행/셀 내부 분할 단위에서 같은 계열의 과대 분할이 생기는지 확인한다.

### 후보 C — `hide_empty_line` 의미 차이

모든 구역에서 `hide_empty_line`이 활성화되어 있다. 다만 현재 기본 경로 page 1에는 빈 문단 2개가 42.7px로 남아 있다. 이 옵션만으로 33쪽 전체를 설명하기는 어렵지만, 표 분할 앞뒤의 빈 문단 처리와 함께 누적 오차를 키울 수 있으므로 회귀 확인 대상에 포함한다.

## 중간 구현 결과

현재까지 반영한 수정 후보:

- RowBreak 표의 셀 내부 hard-break 완화를 모든 RowBreak 표로 확대했다.
- 텍스트만 있는 문단의 `height_for_fit` 계산에서 저장 `LINE_SEG` vpos span 을 상한으로 사용했다.
- 저장 vpos 상한과 표 꼬리 보정을 전제로 `LAYOUT_DRIFT_SAFETY_PX`를 0px 로 낮췄다.
- RowBreak 표의 마지막 빈 spacer 행이 소폭 overflow 하는 경우 직전 조각에 흡수하도록 제한 보정을 추가했다.
- 1단 구역 끝의 빈 단나누기 문단은 다음 단/밴드가 없으면 별도 빈 페이지를 만들지 않도록 했다.
- 글앞/글뒤 비-TAC overlay 표 뒤의 빈 guide 문단은 다단 flow 높이를 소비하지 않도록 했다.
- 저장 `LINE_SEG` vpos 상한을 단일단 일반 문단의 flow advance에도 제한적으로 적용했다. 단, RowBreak 표 바로 주변 문단은 기존 보수 누적을 유지한다.
- landscape RowBreak continuation 에서는 반복 머리행이 있는 표의 마지막 빈 spacer/짧은 꼬리 행만 제한적으로 직전 조각에 흡수한다.

페이지 정합 분석:

- Q&A 구간 시작은 현재 PDF 279쪽 대비 rhwp 281쪽으로 +2쪽이고, Q&A 후반은 PDF 307쪽 대비 rhwp 310쪽으로 +3쪽이다.
- `부록` 시작은 PDF 309쪽 대비 rhwp 312쪽으로 +3쪽이다.
- `[별표 4]` 후반 구간은 PDF 369/372쪽 대비 rhwp 373/376쪽으로 +4쪽이다.
- `별지 제8호서식`은 PDF 379쪽 대비 rhwp 384쪽으로 +5쪽이다.
- 최종 발행정보 직전 법령/시행규칙 페이지는 PDF 378~382쪽 대비 rhwp 383~387쪽으로 +5쪽이다.

따라서 HWP 기준 남은 +5쪽은 대략 `구역 10 Q&A +3`, `구역 11 법령 비교 RowBreak 표 +1`, `구역 12 별표/별지 RowBreak 표 +1`로 나누어 추적한다. 짧은 행/whole-row overflow tolerance 를 portrait RowBreak 표까지 넓히면 기존 `issue_rowbreak_chart_overlap` 회귀가 발생하므로, 현재는 landscape 반복 머리행 표로만 제한한다.

## 작업 범위

1. 후반부 RowBreak 표 중심으로 `dump-pages`, `dump`, 필요 시 `export-render-tree`를 비교해 초과 분할이 시작되는 표와 페이지를 확정한다.
2. `TypesetEngine`과 `Paginator`의 표 분할 결과를 같은 표 단위로 비교해 공통 축과 typeset 전용 축을 분리한다.
3. 실제 수정은 RowBreak 표의 행/셀 내부 분할 판단 또는 높이 산식에 한정한다.
4. `hide_empty_line`은 직접 원인으로 확인될 때만 수정한다.
5. 목표는 HWP/HWPX 모두 383쪽 회귀 가드로 고정하는 것이다. 다만 현 단계는 아직 383쪽에 도달하지 않아 샘플 fixture 기반 단언 테스트는 추가하지 않는다.

## 리스크

- RowBreak 표 분할은 기존 시각 정합 회귀가 많은 민감 경로다. 한 샘플 페이지 수만 맞추는 넓은 휴리스틱은 피한다.
- PDF는 한컴 2024 산출물이므로 페이지 수 권위는 충분하지만, 세부 시각 판정은 작업지시자 확인이 필요하다.
