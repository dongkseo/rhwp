# 수행계획서 — Task #1700: 표 직후 빈 문단 페이지 배치 보존

- 마일스톤: M100 (v1.0.0)
- 이슈: [edwardkim/rhwp#1700](https://github.com/edwardkim/rhwp/issues/1700)
- 브랜치: `local/task1700` (← `local/devel`)
- 작성일: 2026-06-30

## 1. 배경 / 문제

한컴 한글 2022 정답지 대비 1,000개 표본 페이지·PI 매칭 검증 결과(불일치 68건),
**18건(26%)**이 단일 원인 — **표 직후의 빈 문단이 rhwp 페이지 배치에서 누락**.

- PARA_COUNT off-by-one **12건**: 표 하나짜리 법령 별표 hwp, 전부 `rhwp = 한글 − 1`.
- PI_MISMATCH `rhwp_pNone` **6건**: 표 사이/뒤 빈 문단을 rhwp가 어느 페이지에도 미배치.

증상 상세·IR 근거는 이슈 #1700 및 `output/poc/pi_page_vs_hangul_1000_report.md` 참조.

## 2. 근본원인 (코드 특정)

`src/renderer/pagination/engine.rs:642-683` — 어울림(Square wrap) 표 뒤의 후속 문단이
표의 `(column_start, segment_width)`와 일치하면 **`current_column_wrap_around_paras`
버킷에 넣고 `continue`** 하여 일반 배치 아이템(`current_items`)으로 내보내지 않는다
(표가 차지한 공간이므로 **높이 0 소비**가 의도된 동작).

- `wrap_around_paras` 는 레이아웃(`layout.rs` / `paragraph_layout.rs`의
  `layout_wrap_around_paras`)에서 표 옆에 배치되어 **텍스트가 있으면 시각적으로 렌더링**된다.
- 그러나 **빈 문단**은 가시 글리프가 없고, `dump-pages`/렌더링 쿼리
  (`src/document_core/queries/rendering.rs`)에서 **`pi=` 항목으로 노출되지 않는다.**
- 결과: 문단→페이지 매핑에서 해당 PI가 빠져 한글과 1개씩 어긋남(off-by-one / `rhwp_pNone`).

핵심: **높이 흡수(0 소비)는 옳다.** 다만 그 빈 문단의 **페이지 위치(=표의 페이지)가
배치 산출물에 기록되지 않는 것**이 문제다.

## 3. 목표 / 완료 조건

1. 표 직후(어울림 흡수된) 빈 문단도 한글과 동일하게 **본문 문단으로서 페이지에 등록**되어
   문단→페이지 매핑이 일치한다(`dump-pages`의 `pi=` 목록에 등장).
2. 위 18건이 `verify_pi_page_vs_hangul.py`에서 **MATCH**로 전환.
3. **높이 회귀 없음**: 페이지 수가 기존 대비 변하지 않는다(빈 문단 높이 0 소비 유지).
4. **시각 회귀 없음**: 어울림(텍스트가 표 옆에 흐르는) 정상 케이스의 렌더 결과 불변.

## 4. 영향 범위 / 리스크

- 영향 모듈: `renderer/pagination/engine.rs`(빈 문단 페이지 등록), 필요 시
  `document_core/queries/rendering.rs`(wrap-around 빈 문단을 PI 항목으로 표면화).
- 리스크 R1: 어울림 표 옆 **텍스트** 문단까지 일반 배치로 되돌리면 높이 이중계상 →
  페이지 수 회귀. → **빈 문단에 한정**, 높이 0 소비 유지로 차단.
- 리스크 R2: 표가 페이지 경계를 넘는 케이스에서 빈 문단의 귀속 페이지 모호.
  → 표의 시작/끝 페이지 중 한글과 동일한 쪽으로 귀속(구현계획서에서 케이스 분기).
- 리스크 R3: hwp3 variant 등 다른 경로 영향. → hwp3 전용 분기 추가 금지(CLAUDE.md),
  공통 경로에서만 처리.
