# 수행계획서 — Task #1624

**제목**: footer over-push 정밀 조사·수정 — #1611 vpos 동기화의 +1쪽 부작용
**마일스톤**: M100 · **이슈**: edwardkim/rhwp#1624 · **브랜치**: `local/task1624` (base: `local/task1618`)
**전사**: #1611(footer Page+Bottom page-fit)이 −1쪽을 줄였으나 일부 +1쪽 over-push 유발(코퍼스 ~3%).

## 1. 근본 원인 (조사 확정)
#1611 의 `sync_h = max(cur_h, target_y)` 가 footer 의 stored vpos 로 cur_h 를 끌어올린다.
본문이 짧은데 footer vpos 가 page-bottom 앵커/누적 노이즈로 매우 큰 경우(예: vpos 987~1628px
vs 본문 cur_h 28~602px), 본문 직후에 들어갈 footer 를 spurious 하게 다음 쪽으로 민다.

판별자(진단 RHWP_DBG_FOOTER): **gap = vpos − cur_h**.
- 해소된 −1 케이스(sync 정당): gap ≤ +13px (footer 가 본문 직후).
- over-push +1 케이스: gap 959~1026px (vpos 가 본문과 동떨어진 앵커 노이즈).

## 2. 수정
vpos 가 흐름을 plausibly 따를 때만 동기화: `target_y <= cur_h + block_height` 이면
`sync_h = max(cur_h, target_y)`, 아니면 `sync_h = cur_h`(흐름 위치 유지). gap 이 footer 한 개
높이를 초과하면 vpos 무시.
