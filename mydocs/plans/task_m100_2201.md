# Task M100 #2201 수행 계획 - PR review 권한별 반영 경로 정리

## 이슈

- [https://github.com/edwardkim/rhwp/issues/2201](https://github.com/edwardkim/rhwp/issues/2201)

## 목표

`mydocs/manual/pr_review_workflow.md`에서 maintainer와 collaborator의 GitHub 권한 차이를 실제 반영 방식에
연결하고, GitHub review/후속 코멘트의 이슈·PR 참조를 명시적인 URL Markdown 링크로 통일한다.

## 변경 범위

- maintainer/admin, collaborator/write, contributor 권한 기준 정의
- 메인터너 운영 기록의 `devel` 직접 반영 경로 추가
- collaborator 및 코드·테스트·workflow·golden 변경의 PR/CI 경로 명시
- `pr_N_review_impl.md` 선택 작성 기준 정리
- merge 시뮬레이션 방향과 변경 범위별 검증 매트릭스 정정
- golden/baseline 반영 경로 정정
- GitHub review/PR/issue 코멘트의 `[URL](URL)` 링크 규칙 및 예시 추가

## 제외 범위

- `Admin Merge` 절과 `--admin` 사용 정책은 변경하지 않는다.
- 애플리케이션 소스, 테스트, CI workflow는 변경하지 않는다.

## 검증

- `git diff --check`
- Markdown fenced code block 짝수 여부 확인
- `Admin Merge` 절이 `upstream/devel` 기준 원문과 동일한지 확인
- 변경 파일이 `mydocs/**` 문서로만 제한되는지 확인
