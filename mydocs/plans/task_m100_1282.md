## 배경

PR #1279에서 글상자/셀 중첩 그림의 이동, 회전, 리사이즈 실패 경로와 HWP 저장 시 회전 그림 matrix 계약을 보강했다.
다만 `samples/ta-pic-001-r.hwp` 기준으로 한컴편집기의 조작 UX와 비교하면 다음 후속 문제가 남아 있다.

- 표 안 그림 회전/리사이즈 후 회전 외곽이 셀 높이를 초과할 때 한컴처럼 셀 높이가 자동 증가해야 한다.
- 회전된 이미지 크기를 줄일 때 rhwp-studio에서 이미지 중심과 크기 반영이 불안정한 현상을 정리해야 한다.

## 문제 인식

현재 저장 계약은 대체로 다음 구조로 고정되어 있다.

- `common.width/height`: 회전 후 외곽 박스
- `shape_attr.current_width/current_height`: 회전 전 실제 표시 크기
- HWP5 저장: SHAPE_COMPONENT rendering matrix 생성

Issue #1282는 이 저장 계약 자체보다, 편집 중 속성 적용 시 주변 표 모델과 드래그 계산이 같은 의미를 공유하지 못하는 문제로 본다.

특히 셀 내부 그림은 `cellPath` 기반 setter를 통해 속성이 적용되지만, 그림의 회전 외곽이 커졌을 때 소유 셀/행/표 높이까지 같이 조정하는 정책이 없다.
또한 드래그 리사이즈는 화면상의 bbox와 저장 속성의 `common`/`curSz` 의미가 섞일 수 있어, 회전된 그림을 줄일 때 중심이 흔들리거나 예비 테두리와 실제 반영이 어긋날 수 있다.

## 목표

1. 회전된 셀 내부 그림의 bbox가 셀 내부 높이를 초과하면 셀 높이와 표 전체 높이를 한컴편집기처럼 증가시킨다.
2. 회전 그림 리사이즈 계산에서 화면 bbox, 회전 전 실제 크기, 저장 offset의 의미를 분리한다.
3. HWP/HWPX 샘플의 기존 회전 저장 계약을 깨지 않는다.
4. rhwp-studio 조작 경로에서 리사이즈 예비 테두리, 실제 이미지, undo/redo가 같은 결과를 보이게 한다.
5. Rust 회귀 테스트와 Studio E2E/시각 검토 자료를 함께 남긴다.

## 단계 계획

### Stage 1 — 기준값과 재현 범위 확정

- `ta-pic-001-r`의 셀, 표, 그림 크기/offset/rotation 값을 덤프한다.
- 현재 `setCellPicturePropertiesByPath`로 rotation/resize를 적용했을 때 셀 높이와 표 높이가 어떻게 남는지 기록한다.
- 한컴편집기 시각 판정이 필요한 조작 목록을 문서화한다.

### Stage 2 — 셀 높이 자동 증가 모델

- 셀 내부 picture 속성 변경 후 소유 셀의 필요 높이를 계산한다.
- 필요 높이가 현재 셀 높이보다 클 때만 증가시킨다.
- 조정 후 `Table::update_ctrl_dimensions`와 pagination/cache invalidation을 유지한다.

### Stage 3 — 회전 리사이즈 중심 안정화

- 드래그 계산에서 page bbox와 저장 offset delta를 분리한다.
- 회전 그림의 resize는 `common bbox`와 `curSz` 스케일 의미를 유지한다.
- 예비 테두리와 실제 반영 결과가 같은 좌표계로 보이도록 정리한다.

### Stage 4 — 회귀 테스트와 시각 검토 자료

- Rust 테스트: 회전된 셀 내부 그림 resize/rotation 후 cell/table height 증가 및 저장 계약 보존.
- Studio E2E: `ta-pic-001-r` 열기, 셀 내부 그림 resize/rotation 조작, bbox/속성/undo 검증.
- 필요 시 headless screenshot을 생성해 작업지시자 시각 판정을 받는다.
