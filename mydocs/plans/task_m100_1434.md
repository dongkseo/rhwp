# Task M100 #1434 수행계획서 — 누름틀 안내문(Direction) 한컴 바인딩 결함

- 이슈: #1434 "누름틀 안내문(Direction)이 한컴 에디터에서 바인딩되지 않음 — HWP 저장 호환 결함"
- 마일스톤: M100 (v1.0.0)
- 브랜치: `local/task1434` (`local/devel` 분기)
- 작성일: 2026-06-19

## 1. 현상

rhwp-studio에서 누름틀(ClickHere) + 안내문 지정 후 HWP 저장 → rhwp-studio는 정상,
**한컴 에디터는 안내문 미바인딩**(빈 누름틀). rhwp 자기 정합은 OK(자기 검증 ≠ 한컴 호환).

## 2. 근본 원인 진단 (한컴 권위 샘플 대조 완료)

한컴 원본 누름틀 command (`samples/field-01.hwp`, `form-01.hwp`):

```
Clickhere:set:48:Direction:wstring:6:여기에 입력 HelpState:wstring:0:  
Clickhere:set:47:Direction:wstring:5:제목 입력 HelpState:wstring:0:  
```

분석 (두 케이스 교차검증):
- inner = `Direction:wstring:{gl}:{guide} HelpState:wstring:{ml}:{memo}  ` — HelpState 값 뒤 **공백 2개**(구분 1 + trailing 1).
- **`set` 길이 = inner_len − 1** (마지막 trailing 공백 제외). 여기에 입력: inner 49 → set 48. 제목 입력: inner 48 → set 47.
- **command에 `Name:` 키 없음** — 이름은 별도 CTRL_DATA 레코드(0x57, 0x021b magic)에 저장.

rhwp `Field::build_clickhere_command` (`src/model/control.rs:313`) 의 어긋남 3가지:

| # | 항목 | 한컴 정답 | rhwp 현재 |
|---|------|----------|-----------|
| ① | Name 키 | command에 **없음**(CTRL_DATA에만) | name 있으면 `Name:wstring:{n}:{name}` **추가** |
| ② | trailing 공백 | HelpState 뒤 **2개** | **1개** |
| ③ | set 길이 | inner_len **− 1** | inner_len 전체 |

이름이 command에 끼어들고 set 길이가 어긋나면, 한컴 command 파서가 Direction 범위를
잘못 잘라 **안내문 바인딩 실패**. 이름은 이미 CTRL_DATA에 있으므로 command의 Name은
불필요·유해. (serializer `src/serializer/control.rs:140~` 가 CTRL_DATA 이름을 이미 방출.)

## 3. 수정 방향 (구현계획서에서 확정)

`build_clickhere_command` 를 한컴 정답 포맷으로 정정:
- Name 키 제거 (이름은 CTRL_DATA 전담).
- HelpState 뒤 trailing 공백 2개.
- `set` 길이 = inner_len − 1.

호출처(`field_query.rs:1247`, `wasm_api.rs:4045`)는 시그니처 유지하되 name 인자가
command에 안 들어가게(또는 시그니처 단순화). guide/memo 추출(`guide_text`/`memo_text`)
회귀 없음 확인.

## 4. 목표와 완료 조건

1. rhwp 저장 누름틀 command가 한컴 정답 포맷과 바이트 동형 (set 길이·공백·Name 부재).
2. 회귀 가드 단위 테스트 (한컴 권위 케이스 2종 + name 비포함 검증).
3. **한컴 2020/2022 편집기에서 안내문 바인딩 판정 통과** (작업지시자 Windows 환경).
4. 기존 누름틀 파싱(guide_text/memo_text/field_name) 회귀 0, 전체 테스트 그린.

## 5. 단계 개요 (3단계 예정 — 구현계획서에서 확정)

1. `build_clickhere_command` 정정 + 회귀 가드 단위 테스트 (한컴 케이스 대조).
2. 호출처 정합 + 저장 후 재파싱 정합(guide/memo/name 보존) + 전수/CI 검증.
3. 한컴 판정용 샘플 산출(작업지시자 Windows 한컴 검증) + 문서·보고서.

## 6. 위험과 대응

| 위험 | 대응 |
|------|------|
| Name 제거가 기존 파싱(field_name) 회귀 | field_name은 CTRL_DATA name 우선이라 영향 적음. 재파싱 정합 테스트로 봉인 |
| set 길이 규칙이 케이스별 상이 | 한컴 2케이스 교차검증 완료(inner−1 일관). 추가 한컴 샘플로 재확인 |
| 자기 검증 ≠ 한컴 호환 | **반드시 작업지시자 Windows 한컴 편집기 바인딩 판정** 게이트 |

## 7. 검증 방법

- 단위 테스트: 한컴 권위 command 2종 바이트 동형 + name 비포함
- 저장→재파싱 정합: guide/memo/name 보존
- `cargo test --profile release-test --tests` + fmt + clippy
- **한컴 편집기 바인딩 판정** (Windows 정답지)
