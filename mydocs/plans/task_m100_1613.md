# Task M100 #1613 수행계획서 — rhwp-studio 저장 출력 포맷(HWP/HWPX) 선택 메뉴

- 이슈: #1613 "rhwp-studio: 저장 시 출력 포맷(HWP/HWPX) 사용자 선택 메뉴"
- 마일스톤: M100 (v1.0.0)
- 브랜치: `local/task1613`
- 작성일: 2026-06-28

## 1. 배경

#1532(PR #1533)로 HWPX 직접 저장이 활성화되어, 현재 `saveCurrentDocument`/`file:save-as` 는
`getSourceFormat()` 기준 출처 포맷으로만 저장한다(HWPX→exportHwpx, HWP→exportHwp). 사용자가
출력 포맷을 직접 고를 수 없다.

WASM API(`exportHwp()`/`exportHwpx()`)는 출처 무관 양방향 export 를 이미 지원한다(HWP 문서에서
HWPX 내보내기, HWPX 문서에서 HWP 내보내기 모두 가능). studio UI/명령 레이어에 포맷 선택 저장만
추가하면 된다.

## 2. 사실 확인 (소스 조사 완료)

| 항목 | 확인 |
|---|---|
| WASM 양방향 export | `wasm-bridge.ts` `exportHwp()`/`exportHwpx()` 존재, 출처 무관 호출 가능 |
| 현재 저장 분기 | `file.ts` `file:save-as` 가 `isHwpx = getSourceFormat()==='hwpx'` 로 export·파일명·MIME 결정 |
| 메뉴 정의 | `index.html` `.md-item data-cmd="file:save"/"file:save-as"` (line 26-27), menu-bar.ts 가 data-cmd dispatch |
| 저장 헬퍼 | `saveDocumentToFileSystem`(FS Access + 폴백), `saveFileNameFor`/`saveBaseNameFor`(포맷별 확장자) |

## 3. 목표

저장 시 사용자가 출력 포맷(HWP/HWPX)을 명시 선택할 수 있게 한다.

- HWPX 문서: 기본 HWPX 저장(현행) + 메뉴로 HWP 저장 선택.
- HWP 문서: 기본 HWP 저장(현행) + 메뉴로 HWPX 저장 선택.

## 4. 범위

- **포함** (rhwp-studio TS/HTML 전용):
  - `index.html` 파일 메뉴에 포맷 지정 저장 항목 추가("HWP 형식으로 저장" / "HWPX 형식으로 저장").
  - `command/commands/file.ts` 에 포맷 명시 저장 경로 추가(출처가 아닌 선택 포맷으로 export·확장자·MIME).
  - 명령 등록(`file:save-as-hwp` / `file:save-as-hwpx`, 네이밍 구현 시 확정) + 메뉴 표시.
  - 기존 `file:save`/`file:save-as` 출처 인식 기본 저장은 유지.
- **제외**: WASM/Rust 변경 없음(기존 API 재사용). HWP↔HWPX 변환 충실도 개선은 별개(한컴 호환은
  작업지시자 시각 판정 권위).

## 5. 검증 기준

1. HWP 문서 열기 → "HWPX 형식으로 저장" → `.hwpx`(application/hwp+zip, PK 매직) 산출.
2. HWPX 문서 열기 → "HWP 형식으로 저장" → `.hwp`(application/x-hwp) 산출.
3. 기본 저장(`file:save`/`file:save-as`)은 출처 포맷 유지(현행 회귀 없음).
4. studio tsc 에러 0, npm test 통과, e2e 포맷 선택 저장 검증(가능 시).
