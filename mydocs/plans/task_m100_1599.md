# 수행계획서 — Task M100 #1599

- edwardkim/rhwp#1599 — 표의 중심선이 표시되지 않음
- 브랜치: `task_m100_1599_table_center_line`
- 기준 브랜치: `upstream/devel`

## 배경

사용자가 제공한 예시 문서는 WBS 형태의 표에서 셀 중심선을 사용한다. 현재 rhwp 렌더링은
`BorderFill.attr`의 slash/backSlash 대각선 비트는 처리하지만 중심선 비트는 렌더링하지 않아,
한컴 정답 PDF와 비교할 때 셀 안쪽 중앙선이 누락된다.

참고 샘플:

- `samples/추진일정.hwp`
- `samples/추진일정.hwpx`
- `pdf/추진일정-2024.pdf`

## 범위

1. HWP5/HWPX `BorderFill` 중심선 속성 수용 상태 확인
2. 셀 대각선 렌더링 경로에서 중심선 렌더링 추가
3. HWPX `centerLine` 속성이 누락되어 있으면 `BorderFill.attr` 중심선 비트로 보존
4. 단위 테스트와 샘플 SVG 생성으로 회귀 검증
5. 최종 단계에서 WASM 빌드 후 시각 판단 대기

## 제외

- 표 레이아웃, 행/열 크기, 페이지네이션 보정
- 중심선 UI 편집 기능
- 중심선 외 테두리/채우기 저장 계약 변경

## 검증 계획

1. `cargo test --lib` 중 관련 parser/renderer/serializer 테스트
2. `cargo fmt --check`
3. 샘플 `samples/추진일정.hwp`와 `samples/추진일정.hwpx` SVG 내보내기 후 중심선 stroke 생성 확인
4. `wasm-pack build --target web --out-dir pkg`
5. 정답 PDF와 출력 SVG를 시각 비교 대상으로 보관하고 사용자 시각 판단을 기다린다.
